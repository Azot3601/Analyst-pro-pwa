# TASK_DESIGN_GUIDE

## Структура задачи

Каждая задача содержит:

- бизнес-контекст;
- зачем это аналитику;
- исходные артефакты;
- что нужно сделать;
- критерий правильности;
- многоуровневые подсказки;
- объяснение;
- частые ошибки;
- связанные знания.

## Подсказки

1. Лёгкий намёк.
2. Направление решения.
3. Структурная подсказка.
4. Почти готовое решение.
5. Разбор после ответа.

## SQL Quest

SQL-задачи проектируются по формуле:

`сюжетная причина → SQL-действие → проверяемый результат → ошибка с объяснением → связанное знание → награда`

Правила:

- SQL и аналитическое мышление важнее сюжета.
- Сюжет объясняет, зачем нужен запрос, но не заменяет бизнес-контекст.
- `trial` проверяет точность концепта.
- `case` добавляет риск неверной трактовки, ловушку данных или интеграционный смысл.
- Ошибка должна объяснять разницу по строкам, колонкам и вероятной причине.
- Каждая задача должна вести к knowledge node, который помогает исправить ошибку.
