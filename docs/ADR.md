# ADR

## ADR-001: Dark-first интерфейс

Статус: принято.

Решение: основной интерфейс dark-first, светлая тема доступна как настройка.

Причина: пользователь явно попросил ночной комфорт и современный уникальный дизайн.

## ADR-002: CodeMirror вместо Monaco

Статус: принято.

Решение: редакторы SQL/JSON/OpenAPI строятся на CodeMirror.

Причина: меньше bundle, проще PWA/offline, достаточно возможностей для тренажёров.

## ADR-003: React Flow для графа знаний

Статус: принято.

Решение: knowledge graph реализуется через React Flow.

Причина: интерактивные узлы, связи, pan/zoom и доступная интеграция в React.

## ADR-004: sql.js для SQL-тренажёра

Статус: принято.

Решение: SQL выполняется в браузере через sql.js.

Причина: локальная база, offline-first, отсутствие backend.
