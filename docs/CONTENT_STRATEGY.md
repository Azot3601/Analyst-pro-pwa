# CONTENT_STRATEGY

Контент строится вокруг практики системного аналитика: бизнес-контекст, артефакты, критерий правильности, подсказки, объяснение, частые ошибки и связь с базой знаний.

## Минимальный полный объём

- 30 SQL-задач.
- 25 REST/API-задач.
- 25 JSON/OpenAPI-задач.
- 15 интеграционных кейсов.
- 80+ узлов базы знаний.
- 20+ инструментов и чеклистов.

## Подход

- Сначала качественная репрезентативная часть по всем доменам.
- Затем расширение через однотипные типизированные data files.
- Каждый материал должен быть валидируемым и связанным с knowledge graph.

## SQL-траектория

SQL больше не должен выглядеть как набор независимых карточек. Базовый поток строится как SQL Quest Mode, подробнее в `docs/SQL_QUEST_MODE.md`.

Текущий объём SQL Quest:

- 8 глав;
- 24 задачи;
- два пути: `trial` и `case`;
- 10 точных SQL knowledge links;
- XP/ranks и persistent progress.

Каждый шаг обязан показывать таблицы, связи, бизнес-контекст, зачем это аналитику, критерии успеха, подсказки, табличный результат, частые ошибки и диагностический разбор mismatch.
