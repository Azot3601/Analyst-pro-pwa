# CONTENT_STRATEGY

Контент строится вокруг практики системного аналитика: бизнес-контекст, артефакты, критерий правильности, подсказки, объяснение, частые ошибки и связь с базой знаний.

## Минимальный полный объём

- 30 SQL-задач.
- 25 REST/API-задач.
- 25 JSON/OpenAPI-задач.
- 15 интеграционных кейсов.
- 80+ узлов базы знаний.
- 20+ инструментов и чеклистов.

## Подход

- Сначала качественная репрезентативная часть по всем доменам.
- Затем расширение через однотипные типизированные data files.
- Каждый материал должен быть валидируемым и связанным с knowledge graph.

## SQL-траектория

SQL больше не должен выглядеть как набор независимых карточек. Базовый поток строится как квест:

1. Выбрать нужные колонки через `SELECT`.
2. Отфильтровать строки через `WHERE`.
3. Найти приоритетную строку через `ORDER BY` и `LIMIT`.
4. Добавить связанный контекст через `JOIN`.
5. Сформировать показатель через `GROUP BY`, `COUNT`, `SUM`.
6. Найти дубли через `HAVING`.
7. Найти отсутствие связанного факта через `LEFT JOIN` и `IS NULL`.
8. Классифицировать состояние через `CASE`.
9. Использовать подзапрос для динамического порога.
10. Использовать `CTE` для сверки заказа и платежей.

Каждый шаг обязан показывать таблицы, связи, бизнес-контекст, зачем это аналитику, критерии успеха, подсказки, табличный результат и частые ошибки.
